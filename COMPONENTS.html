KOLFF Design System

Comprehensive design guidelines and component library

The KOLFF Design System is a comprehensive set of design principles, patterns, and reusable components designed to create consistent, accessible, and professional digital experiences. This system ensures coherence across all KOLFF platforms and products.

✓

Design Principles

Core values guiding all design decisions

Clarity
Every element has a clear purpose. Visual hierarchy guides users naturally through content.
Consistency
Unified patterns and components create a predictable, familiar experience across platforms.
Accessibility
WCAG 2.1 AA compliant. Inclusive design for all users regardless of ability.
Performance
Optimized for speed and efficiency. Components load instantly and respond immediately.
Scalability
Built to grow. System scales from single components to enterprise-level applications.
Professionalism
Premium aesthetic that conveys trust and authority in the energy infrastructure sector.

Accessibility

WCAG 2.1 AA compliance standards

Color Contrast
All text meets WCAG AA standards with minimum 4.5:1 contrast ratio for normal text and 3:1 for large text.
Keyboard Navigation
All interactive elements are keyboard accessible. Visible focus indicators guide keyboard navigation.
Focus Management
Clear, visible focus states on all interactive elements. Tab order follows logical document flow.
Semantic HTML
Proper semantic markup ensures compatibility with screen readers and assistive technologies.
○

Best Practices

Guidelines for component usage

Do: Clear Labels
Always provide clear, descriptive labels for form fields and interactive elements.
Don't: Ambiguous Actions
Avoid vague button text like "Submit" without context. Use "Save Changes" instead.
Do: Consistent Spacing
Use the spacing system consistently throughout your layouts to maintain visual harmony.
Don't: Excessive Colors
Limit your color palette to the defined system. Avoid custom colors that break consistency.
Do: Provide Feedback
Use alerts and notifications to provide clear feedback on user actions and system states.
Don't: Disable Buttons Without Reason
Only disable buttons when there's a specific reason. Provide helper text explaining why.
