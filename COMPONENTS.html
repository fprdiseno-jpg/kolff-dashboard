Design System v2.0

Professional component library and design documentation for KOLFF digital products

This Design System provides a comprehensive set of components, patterns, and guidelines to create consistent, accessible, and professional digital experiences across all KOLFF platforms. Built on modern design principles and accessibility standards (WCAG 2.1 AA).

Design Principles

Core values guiding all design decisions

Clarity
Every element serves a clear purpose. Visual hierarchy guides users intuitively through content and interactions.
Consistency
Unified patterns and components create predictable, familiar experiences across all platforms and devices.
Accessibility
Inclusive design for all users. WCAG 2.1 AA compliant with keyboard navigation, screen reader support, and color contrast.
Performance
Optimized for speed and efficiency. Components load instantly and respond immediately to user actions.
Scalability
System designed to grow. Scales seamlessly from single components to enterprise-level applications.
Professionalism
Premium aesthetic that conveys trust, authority, and expertise in energy infrastructure solutions.

Accessibility

WCAG 2.1 AA compliance standards

Color Contrast
All text meets WCAG AA standards: 4.5:1 contrast ratio for normal text, 3:1 for large text and UI components.
Keyboard Navigation
All interactive elements are fully keyboard accessible. Tab order follows logical document flow.
Focus Indicators
Clear, visible focus states on all interactive elements. Easily distinguishable from surrounding content.
Screen Reader Support
Semantic HTML and ARIA attributes ensure compatibility with assistive technologies.
Text Sizing
Text can be resized up to 200% without loss of functionality or readability.
Motion & Animation
Animations respect user preferences (prefers-reduced-motion) and don't cause seizures.

Best Practices & Guidelines

Do's and Don'ts for component usage

Do's

  • ✓ Use clear, descriptive labels on form fields and buttons to guide users.
  • ✓ Provide feedback for all user actions (loading states, success messages, errors).
  • ✓ Maintain consistent spacing using the design token scale throughout.
  • ✓ Test with keyboard navigation to ensure all functionality is accessible.
  • ✓ Use semantic HTML for better accessibility and SEO.

Don'ts

  • ✕ Don't use custom colors outside the defined system. Maintain consistency.
  • ✕ Don't disable buttons without reason. Provide helper text explaining why.
  • ✕ Don't rely solely on color to convey information (colorblind accessibility).
  • ✕ Don't create new components when existing ones can be adapted.
  • ✕ Don't ignore mobile responsiveness. All components must work on small screens.
